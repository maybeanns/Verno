# 13-02: Verno CI Pipeline

## Goal
Implement a robust GitHub Actions workflow that executes Lint, Build, and Test for the VS Code extension natively, securing the SDLC pipeline's CI foundation.

## Tasks

### 1. Scaffold CI Action
<read_first>
- package.json
</read_first>

<action>
Create `.github/workflows/ci.yml` based on the standard `@vscode/test-electron` test structure.
Ensure it includes:
1. `name: Verno CI`
2. `on: [push, pull_request]`
3. `jobs: build: runs-on: ubuntu-latest` setup.
4. Steps to `actions/checkout@v4`, `actions/setup-node@v4` with Node 18x.
5. Steps to `npm ci`, `npm run compile`.
6. A step to run `npm test` wrapped inside xvfb so the extension host boots without an actual display monitor (using `xvfb-run -a npm test` on linux).
</action>

<acceptance_criteria>
- `.github/workflows/ci.yml` exists and triggers on `push` and `pull_request`.
- Node setup is `v18`.
- It executes `npm run compile` and `xvfb-run -a npm test`.
</acceptance_criteria>

## Verification
- GitHub action YAML is correctly formatted.
- `package.json` has an `npm test` script ready for test-electron execution.
- Optionally run `act` or visually review the `.yml` syntax limits.
