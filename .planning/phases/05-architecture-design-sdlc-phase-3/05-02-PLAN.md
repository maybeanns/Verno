# Plan 05-02: Mermaid Diagrams (ARC-02)

## Objective
Extend `ArchitectAgent` to generate standard Mermaid diagrams (Sequence, Component, ER) as part of its output, ensuring architectural visualization within the VSCode markdown preview.

## Steps
1. **Prompt Engineering - Diagrams**:
   - Instruct the LLM in `ArchitectAgent.ts` to output standard Mermaid diagrams (using ```mermaid code fences) when writing the architecture document.
   - Require a minimum of a high-level Component diagram (or System Context).
   - Require a Sequence diagram if interactions between multiple services are detected.
2. **Output Handling**:
   - The buffer string streamed from the LLM will naturally contain the Mermaid fences. 
   - No separate files handles are needed for this requirement since VSCode natively parses them in `ARCHITECTURE.md`.
3. **Safety Fallback**:
   - Ensure the prompt strictly warns against unsupported syntax (such as complex flowcharts requiring newer unbundled mermaid versions). Stick to standard graph TD, sequenceDiagram, erDiagram.
