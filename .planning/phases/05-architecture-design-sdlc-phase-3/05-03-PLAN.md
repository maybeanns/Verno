# Plan 05-03: OpenAPI Contracts (ARC-03)

## Objective
Extend `ArchitectAgent` to generate OpenAPI 3.1 YAML contracts when the underlying requirements specify a RESTful API.

## Steps
1. **API Requirement Detection**:
   - Update `ArchitectAgent.ts` to scan the previous outputs (`analyst` or `pm`) for keywords like API, HTTP, REST, or endpoints.
   - Alternatively, use LLM logic during the architecture generation to determine if an API specification is necessary.
2. **Dedicated Contract Generation**:
   - If an API contract is deemed necessary, trigger a secondary call to `llmService.generate` (or `streamGenerate`) specifically tailored to output raw OpenAPI 3.1 YAML.
   - The prompt for this call should include the analyzed architecture context and demand ONLY valid YAML without markdown boilerplate or conversation text.
3. **File Writing**:
   - Capture the output and save it to a conventional location, typically `docs/api/openapi.yaml` or `api/openapi.yaml`.
   - Use `fileService.createFile` to execute the write operation.
4. **Agent Integration**:
   - Add completion task log: `Completed OpenAPI 3.1 specification generation`.
   - Record the file change in `changeTracker`.
