# Plan 05-01: ADR Generation (ARC-01)

## Objective
Extend `ArchitectAgent` to generate Architecture Decision Records (ADRs) when architecting a solution, saving them to `docs/architecture/decisions/` and updating an `ARCHITECTURE.md` index.

## Steps
1. **Update LLM Prompt for ArchitectAgent**:
   - Require Winston to identify key architectural choices and return them in a structured JSON block or distinctly marker format within the response.
   - Format must follow MADR (Markdown Architecture Decision Records): Context, Decision, Consequences.
2. **Post-Process Output**:
   - Extract the MADR content from the LLM's response.
   - Dynamically create the `docs/architecture/decisions/` directory if it does not exist using `fileService`.
   - Iterate over each decision and generate sequentially numbered markdown files (e.g., `0001-[slug].md`).
3. **Index Generation**:
   - Build a central `ARCHITECTURE.md` file that summarizes the architecture and provides a table of contents to the individual ADR files.
4. **Agent File Handling Integration**:
   - Use `this.fileService.createFile` for each ADR in the `ArchitectAgent.ts` `execute` method.
   - Record each generated file in `this.changeTracker`.
