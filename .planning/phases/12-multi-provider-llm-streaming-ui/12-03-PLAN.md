# Plan 12-03: Provider Selector UI + Streaming Markdown

## Goal

Add a UI dropdown to select the active provider and model, passing this selection to the extension backend. Update the chat UI to render streaming markdown into styled HTML efficiently, avoiding flickering while parsing code blocks.

## Threat Model

<threat_model>
- **Threat**: Cross-Site Scripting (XSS) via markdown rendering of LLM output.
  **Mitigation**: Use `marked.js` with HTML sanitization enabled (or a strict CSP in the webview) to ensure no `<script>` tags from the LLM can execute.
- **Threat**: Webview accepts arbitrary model commands from UI.
  **Mitigation**: The `processInputSubmit` message validation ensures only known model identifiers are processed.
- **ASVS**: Webview messages implement a strict type allowlist (`SIDEBAR_ALLOWED_TYPES`).
</threat_model>
