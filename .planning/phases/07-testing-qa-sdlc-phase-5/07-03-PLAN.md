# Plan 07-03: Coverage Badge UI (TST-03)

## Objective
Present live test coverage and passing/failing state inside the Verno Sidebar using dynamic UI badges, closing the loop between test auto-generation and user visibility.

## Steps
1. **Istanbul/Jest Integration**:
   - `QAEngineerAgent` enforces that the package setup runs unit tests with an Istanbul coverage reporter (`jest --coverage` or `vitest run --coverage`).
2. **VSCode Sidebar Backend (`SidebarProvider.ts`)**:
   - Create a filesystem watcher hooking onto the `coverage/coverage-summary.json` file inside the workspace root.
   - Upon change, read the JSON values for Statements, Lines, Branches, and Functions.
3. **Webview UI (`media/main.js` / HTML)**:
   - Introduce a new `<div id="coverage-badge">` below the Progress Indicator.
   - Design dynamic CSS: High (>80%) renders green, Med (>50%) renders orange, Low renders red.

## Files Modified
- `src/agents/BMAD/QAEngineerAgent.ts` (tooling rules)
- `src/webview/SidebarProvider.ts`
- `media/main.js` and `media/style.css`

## Verification
- Running tests that emit `coverage-summary.json` visibly shifts the badge state inside the active VSCode sidebar without requiring a manual refresh.
