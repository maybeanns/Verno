# Phase 7: Testing & QA (SDLC Phase 5) - Context

**Gathered:** 2026-04-14
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement automated unit testing stubs, logic-aware test generation, Playwright E2E scaffolding based on PRDs, and a coverage visualization system within the Verno extension. This also includes the core `MockLLMService` for self-testing the Verno agents.

</domain>

<decisions>
## Implementation Decisions

### Unit Testing Framework
- **D-01:** Use **Jest** as the primary unit testing framework.
- **D-02:** Standardize on `.test.ts` naming convention for all generated test files.

### Test Generation Strategy
- **D-03:** AI-Driven Logic-Aware tests.
- **D-04:** The `TestGeneratorAgent` must analyze function bodies to generate meaningful assertions, boundary tests, and edge cases, rather than just empty scaffolding.

### E2E Testing with Playwright
- **D-05:** Context-Driven Scripts.
- **D-06:** Playwright scaffolds must be generated by mapping User Flows and Acceptance Criteria from the existing `REQUIREMENTS.md` or PRD artifacts into executable scripts.

### Observability
- **D-07:** Sidebar Coverage Badge.
- **D-08:** Display coverage percentage in the extension sidebar, updated post-execution of the test runner.

### Mocking & Determinism (TST-01)
- **D-09:** **Record/Replay** MockLLMService.
- **D-10:** Implement a service capable of recording real LLM responses and replaying them based on prompt hashing to ensure deterministic CI/CD runs for Verno's own agents.

</decisions>
