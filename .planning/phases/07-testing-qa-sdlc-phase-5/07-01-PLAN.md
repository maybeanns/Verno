# Plan 07-01: Unit Test Auto-Generator (TST-01)

## Objective
Convert `QAEngineerAgent.ts` from a simple document generator (`QA_PLAN.md`) into a strict test-code generator that parses `FILE:` diffs identical to the developer agent and creates per-file `*.spec.ts` test stubs alongside the code.

## Steps
1. **Prompt Engineering Update**:
   - Instead of asking for test scenarios, explicitly demand output formatted as ````FILE: path/to/file.spec.ts`.
   - Provide the generated source code (`previousOutputs['developer']`) and the Architecture design.
2. **File Output Integration**:
   - Re-use the existing file parsing `parseCodeFiles` logic from `DeveloperAgent` (or extract it to a shared Utility) into `QAEngineerAgent`.
   - Save the raw Markdown logic to `QA_PLAN.md`, but extract the `spec` files to disk explicitly via `FileService`.
3. **Execution Dependency**:
   - If `npm test` exists, automatically run `npm test` post-generation on the newly generated test stubs.

## Files Modified
- `src/agents/BMAD/QAEngineerAgent.ts`
- `src/utils/CodeParser.ts` (newly extracted shared parser, optionally)

## Verification
- Run a pipeline execution and assert that `src/agents/BMAD/QAEngineerAgent.ts` produces physical `.spec.ts` files automatically in the target workspace.
