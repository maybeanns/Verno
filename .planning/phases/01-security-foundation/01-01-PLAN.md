# Phase 1, Plan 1: Secure API Key Storage

<objective>
Migrate all API key handling from `vscode.window.showInputBox` to `vscode.SecretStorage`. Users should only be prompted for their API keys once — keys are then stored securely and retrieved on subsequent sessions automatically.
</objective>

<requirements>
- SEC-01: API keys stored via VSCode SecretStorage, never in plain-text settings or memory-leaked paths
- PRV-03: Provider API keys persisted in SecretStorage
</requirements>

<must_haves>
- [ ] `ConfigService` exposes `getApiKey(provider)` and `storeApiKey(provider, key)` using `vscode.ExtensionContext.secrets`
- [ ] `processUserInput()` in `extension.ts` calls `getApiKey()` first; only prompts if no key stored
- [ ] Stored key used to initialize `GeminiProvider` / `GroqProvider`
- [ ] `ManageAgentsCommand` or a new `verno.clearApiKeys` command allows key deletion
- [ ] `verno.anthropicApiKey` and `verno.groqApiKey` settings entries removed from `package.json` (security: keys must not be in plain-text settings)
</must_haves>

<implementation_steps>

## Step 1 — Extend ConfigService with SecretStorage

In `src/config/ConfigService.ts`, add `setSecretStorage(secrets: vscode.SecretStorage)` method and implement:

```typescript
async getApiKey(provider: 'gemini' | 'groq' | 'anthropic' | 'openai'): Promise<string | undefined>
async storeApiKey(provider: 'gemini' | 'groq' | 'anthropic' | 'openai', key: string): Promise<void>
async deleteApiKey(provider: 'gemini' | 'groq' | 'anthropic' | 'openai'): Promise<void>
```

Secret keys follow the pattern: `verno.apikey.${provider}`

## Step 2 — Wire SecretStorage in extension.ts activation

In `activate()`:
```typescript
configService.setSecretStorage(context.secrets);
```

## Step 3 — Update processUserInput() to use SecretStorage

Replace:
```typescript
apiKey = await vscode.window.showInputBox({ prompt: 'Enter your API key...', password: true });
```

With:
```typescript
apiKey = await configService.getApiKey(detectedProvider);
if (!apiKey) {
    apiKey = await vscode.window.showInputBox({ prompt: `Enter your ${providerName} API key`, password: true });
    if (apiKey) await configService.storeApiKey(detectedProvider, apiKey);
}
```

## Step 4 — Add verno.clearApiKeys command

Register `verno.clearApiKeys` command that:
1. Shows QuickPick of stored providers
2. Calls `configService.deleteApiKey(selected)`
3. Shows confirmation message

## Step 5 — Clean up package.json settings

Remove `verno.anthropicApiKey`, `verno.groqApiKey` from `contributes.configuration.properties`. Add `verno.defaultProvider` setting (enum: gemini/groq/anthropic/openai).

## Step 6 — Update webview to not send API key via postMessage

The `AgentPanel` webview currently sends `apiKey` in messages. After this change:
- Remove `apiKey` field from webview→extension message payloads
- Extension retrieves key from SecretStorage internally
- Webview sends only: `{ type: 'processInput', input: string, mode: string, model: string }`

</implementation_steps>
