# Plan 06-03: Incremental Diff Generation (COD-03)

## Objective
Increase the speed and lower token costs of `DeveloperAgent` by teaching it to only output changed sections via a structured diff format (````diff`) instead of rewriting the entire file contents every time an edit is requested.

## Steps
1. **Developer Agent Edit Prompt Extraction**:
   - In `DeveloperAgent.buildEditPrompt`, instruct the LLM to use a unified diff representation instead of ````EDIT`.
   - Update the expected output format to:
     ```
     ```diff
     FILE: path/to/file.ts
     - old code
     + new code
     ```
     ```
2. **Implement Diff Application Tool**:
   - Replace the simplistic `try ... fileService.updateFile` with a custom patch applicator algorithm.
   - The patch parser will extract `+` and `-` blocks and apply them safely to the existing content using a fuzzy-match line-shifting algorithm (or utilize an existing diff logic library).
3. **Fallback Re-Sync**:
   - If the patch application fails (due to LLM hallucinating surrounding context), intercept the failure and automatically fallback to asking the LLM for full-file generation as a self-healing step.

## Files Modified
- `src/agents/BMAD/DeveloperAgent.ts`
- `src/services/file/FileService.ts` (new `applyPatch` method)
- New file (optional): `src/utils/diffParser.ts`

## Verification
- Pass an edit task measuring token count and latency.
- It should successfully patch a file with a small token footprint.
- A failing patch should trigger the full-file generation fallback smoothly.
