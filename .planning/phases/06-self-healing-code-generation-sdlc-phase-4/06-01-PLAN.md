# Plan 06-01: Self-Healing Code Generation (COD-01)

## Objective
Implement an automatic error-detection and re-generation loop inside `DeveloperAgent.ts`. Instead of just detecting errors and passing them to the feedback service, the agent should automatically retry code generation (up to 3 times) using the compilation errors as context to fix itself.

## Steps
1. **Refactor `execute` loop**:
   - Extract the generation, file writing, and quality checking into a dedicated method or loop constraint inside `DeveloperAgent`.
   - Implement a retry counter (e.g., `MAX_RETRIES = 3`).
2. **Error Context Feedback**:
   - If `runQualityChecks` detects TypeScript (`tsc`) or ESLint errors, capture the exact standard error string (`stderr`) and standard output (`stdout`).
   - If the build fails, append a new PROMPT block: "The previous code generated the following compilation errors. Please fix them, do not generate new functionality, only output the fixed files. ERRORS: [stderr]".
3. **Loop Control**:
   - Send the new contextual prompt back to `llmService.streamGenerate`.
   - Re-parse the resulting files, overwrite the disk, and run `runQualityChecks` again.
   - Stop when `issues.length === 0` or `retries === MAX_RETRIES`.
