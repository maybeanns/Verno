# Phase 2, Plan 1: SDLC-Aware ConversationEngine

<objective>
Upgrade the ConversationEngine from a generic chat assistant to a true SDLC-aware advisor. It should automatically include workspace context in its system prompt, detect what the user is working on (file type, active agent pipeline phase, SDLC stage), and tailor responses accordingly — giving real-time vulnerability alerts, architecture suggestions, or planning guidance.
</objective>

<requirements>
- MOD-01: Conversational mode is a real-time SDLC-aware advisor, not generic chat
</requirements>

<must_haves>
- [ ] `ConversationEngine` builds a **SDLC system prompt** prepended to every conversation, containing: active SDLC phase, recent agent outputs summary, active file language/type, workspace stack (from package.json/pyproject.toml etc.)
- [ ] Conversation mode responds differently based on file context: `.env` → security advice, `*Controller.ts` → API design, `*.test.ts` → testing guidance
- [ ] Users can ask "what vulnerabilities exist in this file?" and get OWASP-grounded answers from the active file content
- [ ] Conversational mode detects if a pending SDLC phase exists (`PlanStateService.hasPendingCodingSteps()`) and surfaces it proactively
- [ ] `WorkspaceIntelligence` service extracts: detected framework, open file, SDLC state, recent changes
</must_haves>

<implementation_steps>

## Step 1 — Create WorkspaceIntelligence service

Create `src/services/workspace/WorkspaceIntelligence.ts`:

```typescript
export interface WorkspaceSnapshot {
    activeFile: string | null;
    activeLanguage: string | null;
    detectedFramework: string | null; // react, express, django, etc.
    sdlcPhase: string | null;         // from PlanStateService
    hasPendingPipeline: boolean;
    recentFiles: string[];
    stackSummary: string;             // "TypeScript + VSCode Extension"
}
```

Methods:
- `getSnapshot(): Promise<WorkspaceSnapshot>` — reads active editor, package.json, PlanStateService
- `getActiveFileContent(maxLines = 100): string` — first N lines of active file
- `detectContext(file: string): 'security' | 'api' | 'test' | 'ui' | 'config' | 'general'`

## Step 2 — Build SDLC system prompt in ConversationEngine

In `src/services/conversationEngine.ts`, inject `WorkspaceIntelligence`:

```typescript
private buildSystemPrompt(snapshot: WorkspaceSnapshot): string {
    return `You are Verno, an SDLC-aware AI coding assistant with expert knowledge of 
software architecture, security (OWASP Top 10), testing, and project planning.

CURRENT WORKSPACE:
- Stack: ${snapshot.stackSummary}
- Active File: ${snapshot.activeFile || 'none'} (${snapshot.activeLanguage || 'unknown'})
- SDLC Phase: ${snapshot.sdlcPhase || 'none active'}
- Pending Pipeline: ${snapshot.hasPendingPipeline ? 'YES — coding phase awaiting execution' : 'none'}

ADVISORY MODE: ${snapshot.activeLanguage === 'dotenv' || snapshot.activeFile?.endsWith('.env') 
    ? 'SECURITY — flag any exposed secrets or insecure patterns'
    : 'GENERAL — provide architecture, code quality, and planning guidance'}

Be concise, specific, and grounded in current best practices.`;
}
```

## Step 3 — Context-aware response templates

Add response shaping based on `detectContext()`:

| Context | Response template |
|---------|-------------------|
| `security` | Lead with OWASP category, risk level, and concrete fix |
| `api` | Lead with REST/GraphQL patterns, status codes, auth |
| `test` | Lead with test coverage, edge cases, mocking strategy |
| `config` | Lead with secret detection, environment parity |
| `general` | Balanced architectural + quality guidance |

## Step 4 — Proactive pipeline surface

On each `think()` call, if `hasPendingPipeline` is true and the user hasn't mentioned the pipeline in 3+ turns:

```
💡 You have a pending coding pipeline from your last planning session.
   Say "run the pipeline" or "continue development" to execute it.
```

## Step 5 — Wire WorkspaceIntelligence into extension.ts

In `activate()`, create and inject `WorkspaceIntelligence` into `ConversationEngine`:
```typescript
const wsIntel = new WorkspaceIntelligence(configService);
brain = new ConversationEngine(wsIntel);
```
