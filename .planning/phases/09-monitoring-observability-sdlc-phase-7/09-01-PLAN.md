# 09-01 PLAN: OpenTelemetry Instrumentation Generator

## Objective
Create `OtelInstrumentationService` + templates to auto-generate OpenTelemetry instrumentation files for generated services, following the Phase 8 scaffold service pattern.

## Files to Create/Modify

### [NEW] `src/services/project/templates/otel-templates.ts`
Export `OTEL_TEMPLATES` record with keys `typescript`, `python`, `generic`.

**TypeScript template** — produces a self-contained `instrumentation.ts`:
- Imports `@opentelemetry/sdk-node`, `@opentelemetry/auto-instrumentations-node`, OTLP HTTP trace/metrics exporters
- Creates `NodeSDK` instance with `serviceName` parameter (injected from `WorkspaceSnapshot.stackSummary`)
- Console exporter as default, OTLP endpoint configurable via `OTEL_EXPORTER_OTLP_ENDPOINT` env var
- Starts the SDK and registers shutdown hooks

**Python template** — produces `instrumentation.py`:
- Uses `opentelemetry.sdk.trace`, `opentelemetry.instrumentation`, OTLP exporter
- Auto-instrumentation for Flask/Django/FastAPI
- `service.name` resource set from env var

**Generic template** — produces `otel-entrypoint.sh`:
- Downloads OTel Java/Go agent via env var
- Wraps the application start command

### [NEW] `src/services/project/OtelInstrumentationService.ts`
- Constructor takes `workspaceRoot: string`
- `generateInstrumentation(snapshot: WorkspaceSnapshot): Promise<string[]>`
  - Detect stack from `snapshot.stackSummary` (same logic as `CiCdScaffoldService`)
  - Write appropriate template to workspace root
  - Return array of created file paths

### [MODIFY] `src/extension.ts`
- Register `verno.generateObservability` command
- Instantiate `OtelInstrumentationService` alongside existing scaffold services
