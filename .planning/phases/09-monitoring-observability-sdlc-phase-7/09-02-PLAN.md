# 09-02 PLAN: Grafana Dashboard + Runbook Generator

## Objective
Create `GrafanaDashboardService` and `RunbookGeneratorService` to produce portable Grafana dashboard JSON and operational runbooks for generated services.

## Files to Create/Modify

### [NEW] `src/services/project/templates/grafana-templates.ts`
Export a `generateGrafanaDashboard(serviceName: string)` function that returns a complete Grafana dashboard JSON object.

Dashboard panels:
1. **HTTP Request Rate** — Prometheus query: `rate(http_requests_total{service="$service"}[5m])`
2. **Error Rate** — `rate(http_requests_total{service="$service", status=~"5.."}[5m])`
3. **Latency p50/p95/p99** — `histogram_quantile(0.95, rate(http_request_duration_seconds_bucket{service="$service"}[5m]))`
4. **Active Connections** — `sum(active_connections{service="$service"})`

Dashboard metadata:
- `title`: `"${serviceName} — Service Dashboard"`
- `uid`: generated from service name
- Templating variable `${DS_PROMETHEUS}` for data source
- Auto-refresh: 30s

### [NEW] `src/services/project/GrafanaDashboardService.ts`
- Constructor takes `workspaceRoot: string`
- `generateDashboard(snapshot: WorkspaceSnapshot): Promise<string[]>`
  - Extract service name from snapshot (project name or directory name)
  - Generate dashboard JSON using template function
  - Write to `.verno/observability/grafana-dashboard.json`
  - Return created file paths

### [NEW] `src/services/project/RunbookGeneratorService.ts`
- Constructor takes `workspaceRoot: string` and `llmService: LLMService`
- `generateRunbook(snapshot: WorkspaceSnapshot): Promise<string[]>`
  - Read `ARCHITECTURE.md` if it exists (from `.verno/` or workspace root)
  - Build a prompt combining:
    - Project stack info from snapshot
    - Architecture context (if available)
    - Built-in failure catalogue (OOM, connection refused, TLS expiry, rate limiting, disk full, dependency timeout)
  - LLM generates a structured Markdown runbook with sections: Detection → Diagnosis → Mitigation → Prevention
  - Write to `.verno/observability/RUNBOOK.md`
  - Return created file paths

### [MODIFY] `src/extension.ts`
- Import and instantiate `GrafanaDashboardService` and `RunbookGeneratorService`
- Wire into the `verno.generateObservability` command (calls all three Phase 9 services)

## Verification
- [ ] Grafana dashboard JSON is valid and importable (correct schema)

- [ ] Runbook includes at least 4 failure scenario sections
- [ ] Both files written to `.verno/observability/`
