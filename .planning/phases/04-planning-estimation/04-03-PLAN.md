# Phase 4, Plan 3: Sprint Auto-Planner

<objective>
Implement a sprint auto-planner that takes a team capacity input (story points per sprint), reads the story point estimates from EstimationAgent, and distributes stories across sprints respecting dependency order. Sprint assignments are persisted and synced to Jira sprints via the JiraSyncService.
</objective>

<requirements>
- PLN-03: Capacity-input sprint distribution + Jira sprint sync
- PLN-02: Dependency-aware ordering (stories that block others go earlier)
</requirements>

<must_haves>
- [ ] `SprintPlannerAgent` distributes Stories across sprints given `capacityPerSprint: number`
- [ ] Sprint ordering respects dependency blocks: blocked stories go in later sprints than their blockers
- [ ] Critical path stories (longest dependency chain) scheduled in sprint 1
- [ ] Sprint plan written to `.verno/sprint-plan.json`
- [ ] Sprint plan synced to Jira sprints via `JiraSyncService` (creates sprints and assigns issues)
- [ ] Sprint plan rendered in sidebar (sprint accordion → stories within each sprint)
</must_haves>

<implementation_steps>

## Step 1 — SprintPlannerAgent

Create `src/agents/BMAD/SprintPlannerAgent.ts`:

```typescript
export interface SprintPlan {
    sprints: Sprint[];
    capacityPerSprint: number;
    totalStoryPoints: number;
    criticalPath: string[];  // Story IDs on critical path
}

export interface Sprint {
    number: number;
    name: string;   // "Sprint 1", etc.
    stories: Story[];
    totalPoints: number;
}
```

Algorithm:
1. Sort stories by dependency order (topological sort on `dependsOn` field)
2. Pack stories into sprints using bin-packing: add stories to current sprint until capacity reached, then start new sprint
3. Flag critical path: longest dependency chain from `DependencyGraph`

## Step 2 — Capacity Input UI

In `SDLCWebviewPanel`, after task breakdown stage, show:
```
⚡ Sprint Planner
Team Capacity: [___] story points per sprint
[Generate Sprint Plan]
```

Post-message to extension: `{ type: 'generateSprintPlan', capacity: number }`

## Step 3 — Jira Sprint Sync

In `JiraSyncService`, add:
```typescript
async createSprint(boardId: string, sprintName: string, goal?: string): Promise<string>
async moveIssueToSprint(issueKey: string, sprintId: string): Promise<void>
```

`SprintPlannerAgent.syncToJira()` creates sprints then bulk-moves issues.

## Step 4 — Sprint Plan sidebar visualization

In `EnhancedSidebarProvider`, add Sprint tab:
- Sprint accordion: Sprint 1 (35/40 SP) → expand → Story cards with points
- Critical path stories highlighted with ⚡ icon
- Progress bar per sprint (done SP vs total SP)

## Step 5 — Persist sprint plan

Write to `.verno/sprint-plan.json`. Load on sidebar open. Re-generate button available.
