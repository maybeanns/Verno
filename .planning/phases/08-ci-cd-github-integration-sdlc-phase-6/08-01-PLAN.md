# 08-01: GitHub Repository Integration

<objective>
Implement a GitHub & Git integration service within Verno that can detect the workspace repository status, identify the active branch, and directly construct commands for Git CLI and GitHub CLI (`gh`). This allows Verno to autonomously commit scaffolded CI/CD files back to the user's repository via a "Direct Commit & Push" workflow.
</objective>

<wave>1</wave>

<requirements>
- Verno must detect if the active workspace is a git repository.
- Verno must identify the active branch.
- Verno must be able to perform `git add`, `git commit`, and `git push` directly to the active branch (D-05: Direct Commit Workflow).
- Ensure a generic interface so that it fails gracefully if `git` or `gh` is not installed or available on the user's PATH.
- Support creation of basic PR commands (e.g., provide user with command script strings for manual execution if direct push fails).
</requirements>

<dependencies>
- `WorkspaceIntelligence.ts` (for workspace root).
- `FileService.ts` (for handling files before Git operations).
</dependencies>

<architecture>
-   `src/services/project/GitService.ts`: A new class responsible for executing generic Git commands within the root workspace directory.
-   `src/services/project/GitHubService.ts`: A thin wrapper leveraging `GitService.ts` to manage direct commits and abstract user commands according to D-05 and D-06 rules.
</architecture>

<implementation_steps>
1. Create `src/services/project/GitService.ts` utilizing `child_process.execFile` for executing Git commands securely (`status`, `branch`, `remote`, `add`, `commit`, `push`).
2. Add workspace Git detection logic (`git rev-parse --is-inside-work-tree`).
3. Create `src/services/project/GitHubService.ts` combining Git operations to execute the defined Direct Commit & Push flow.
4. Expose the `GitHubService.ts` via dependency injection into the Orchestrator/Conversation Agents.
5. Create tests in `test/suite/services/` for the new `GitService`.
</implementation_steps>
