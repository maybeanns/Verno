# 08-03: Containerization & K8s Manifests Generator

<objective>
Provide dynamic creation of Dockerfile, docker-compose.yml, and Kubernetes (Helm) manifests tailored to the user's workspace tech stack, enabling one-step containerization deployments.
</objective>

<wave>2</wave>

<requirements>
- `CIC-02`: Generate `.dockerignore`, `Dockerfile` (multi-stage preferred), and `docker-compose.yml` for local development testing.
- `CIC-03` / `D-03`/`D-04`: Generate production-ready Kubernetes configuration via a standard Helm chart template schema inside the `helm/` directory.
- Align ports and start commands based on `WorkspaceIntelligence` detection outputs (e.g., Node defaults to 3000 or 8080 depending on the framework).
</requirements>

<dependencies>
- `WorkspaceIntelligence.ts`
- `FileService.ts`
</dependencies>

<architecture>
-   `src/services/project/ContainerScaffoldService.ts`: Core service interpreting the detected stack and emitting Container and Helm configurations.
-   `src/services/project/templates/docker-templates.ts`: Pre-built Dockerfile layouts per language.
-   `src/services/project/templates/helm-templates.ts`: Base scaffold logic for `Chart.yaml`, `values.yaml`, and `templates/*.yaml`.
</architecture>

<implementation_steps>
1. Create templates for Docker files (`templates/docker-templates.ts`) covering Node, Python, generic types with multi-stage build best practices.
2. Create Helm templates (`templates/helm-templates.ts`) defining Deployments, Services, and Ingress standard resource YAMLs.
3. Build `ContainerScaffoldService.ts` implementing `generateDockerArtifacts(snapshot)` and `generateHelmArtifacts(projectName, snapshot)`.
4. Integrate into the Phase execution loop to automatically write these to the root workspace.
5. Provide tests mapping Docker and Helm outputs for different `WorkspaceSnapshot` states.
</implementation_steps>
