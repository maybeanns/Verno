# 08-02: CI/CD Scaffold Generator

<objective>
Implement GitHub Actions scaffolding within Verno. Based on the stack detected by `WorkspaceIntelligence`, Verno should dynamically generate `.github/workflows/ci.yml`. This pipeline must be a "Full CI/CD Pipeline" (lint, test, build, Docker build/push).
</objective>

<wave>2</wave>

<requirements>
- Consume stack signals from `WorkspaceIntelligence` (e.g., nodejs, python, generic).
- Map the parsed stack to pre-defined GitHub Action workflow templates (e.g., node pipeline, python pipeline).
- The templates MUST implement a Full CI/CD structure: Job 1 for lint/test/build, Job 2 for Docker build/push.
- Create directories recursively (e.g., `.github/workflows`) if they don't exist.
- Rely on `FileService.ts` for actual disk writing.
</requirements>

<dependencies>
- `WorkspaceIntelligence.ts`
- `FileService.ts`
- 08-01: Needs `GitService` to auto-commit these scaffolds.
</dependencies>

<architecture>
-   `src/services/project/CiCdScaffoldService.ts`: Core service interpreting workspace context and returning structured YAML workflows.
-   `src/services/project/templates/ci-workflows.ts`: Contains string templates/JSON-to-YAML definitions for various stack CI pipes.
</architecture>

<implementation_steps>
1. Create `src/services/project/templates/ci-workflows.ts` with templates for Next.js/React, Express/Node, Python, and a fallback generic template. Incorporate lint, test, build, and docker push steps.
2. Build `CiCdScaffoldService.ts`. Add method `generateCiWorkflow(workspaceSnapshot)` returning generated string contents.
3. Integrate step to write to `.github/workflows/ci.yml` via `FileService`.
4. Inject `CiCdScaffoldService` into the SDLC coding iteration commands, enabling autonomous scaffolding.
5. Provide tests mapping various stacks to verify correct template selection and valid YAML output.
</implementation_steps>
