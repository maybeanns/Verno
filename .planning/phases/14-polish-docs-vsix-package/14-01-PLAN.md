# 14-01: UX Polish + Onboarding Wizard

## Goal
Create a first-run onboarding experience, add keyboard shortcuts, apply the EXT-02/EXT-03 cleanup requirements, and add a mode toggle to the status bar.

## Tasks

### 1. Create Onboarding Welcome Panel

<read_first>
- src/extension.ts
- src/ui/panels/AgentPanel.ts
</read_first>

<action>
Create `src/ui/onboarding/WelcomePanel.ts`:
1. A class `WelcomePanel` with a static `show(context: vscode.ExtensionContext)` method.
2. Uses `vscode.window.createWebviewPanel` with type `'verno.welcome'`.
3. HTML content that explains:
   - The two modes: **💬 Conversational** (always-on pair-programmer) and **🏗️ Development** (full 9-phase SDLC pipeline).
   - A "Set API Key" button that triggers `vscode.commands.executeCommand('verno.clearApiKeys')`.

### 2. Register Keyboard Shortcuts

<read_first>
- package.json (keybindings section — currently absent)
</read_first>

<action>
Add `keybindings` to `package.json` `contributes`:
```json
"keybindings": [
  { "command": "verno.processInput", "key": "ctrl+shift+v", "mac": "cmd+shift+v" },
  { "command": "verno.startRecording", "key": "ctrl+shift+r", "mac": "cmd+shift+r" },
  { "command": "verno.startSDLC", "key": "ctrl+shift+l", "mac": "cmd+shift+l" }
]
```
Note: `ctrl+shift+v` may conflict with paste-as-plain-text. Use `ctrl+shift+;` as fallback if tested and conflicting. Use `ctrl+shift+l` for SDLC (L for Lifecycle).
</action>

<acceptance_criteria>
- `keybindings` array exists in `package.json`.
- At least 3 shortcuts registered.
</acceptance_criteria>
